## üìù Documentation Standards

All code follows these documentation standards:
- Every function has a docstring
- Type hints on all function parameters and returns
- Examples in docstrings where helpful
- Clear parameter descriptions
- Return value documentation

## üîÑ Keeping Documentation Updated

When adding new functions or features:
1. Add docstring to the function
2. Update [API_REFERENCE.md](API_REFERENCE.md) if it's a public API
3. Update [COMMANDS.md](COMMANDS.md) if it's a new command
4. Update [CHANGELOG.md](CHANGELOG.md) with the change
